# Customer Review Script — Invoice Processing Coordinator

> **Purpose:** Structured validation questions for Eric and Shannon
> **Format:** Walk through the app live, ask these questions at each stop

---

## Opening (2 min)

*"This is a working prototype built from our discovery session. The data you see is YOUR actual data — Eric's Main Street Improvements and Shannon's 36th St Bridge. Everything on screen came from your spreadsheets."*

---

## Portfolio Dashboard → Eric

1. Is this the right set of summary metrics for your 14 projects?
2. Would you want to filter by project type (Streets, Parks, Facilities, Surface Water)?
3. Is "On Track / At Risk / Over Budget" the right language for health status?

## Project Detail → Eric (Main Street)

4. Does this budget breakdown match what your spreadsheet shows?
5. Are the 3 contract types correct — Design, CM Services, Construction?
6. For contract supplements: you used to edit the number. Does discrete supplement records make more sense?
7. Is the link-to-signed-document useful? Does it need to open in SharePoint directly?

## Gut-Check Alert → Shannon

8. *"Permitting is 87% spent but only 50% complete"* — is this the kind of alert you'd want?
9. What thresholds feel right? Currently: yellow at +15%, red at +30%.
10. Should these alerts also show on the Portfolio dashboard?

## Invoice Task Breakdown → Shannon

11. When you break an invoice down by task (PM: $5,000, surveying: $3,000), is this the right workflow?
12. Budget totals compute automatically from these breakdowns. Does that match how you'd want it?
13. For grant reimbursement: would you need to filter breakdowns by grant code?

## Import → Both

14. If we import your spreadsheet, does the data look right?
15. What happens when you get an updated spreadsheet — re-import and merge?
16. Are there sheets/tabs we missed in the import?

## Invoice Search → Shannon

17. You look up invoice numbers frequently for grant packages. Is this search sufficient?
18. Would you need to export search results (e.g., all invoices for a specific grant)?

## Pipeline → Both

19. Is Received → Logged → Reviewed → Signed → Paid the right workflow?
20. Would you ever drag an invoice between columns, or is status just tracked elsewhere?

## Export → Eric

21. We can export any project as .xlsx — does the format match what Springbrook needs?
22. Would you need different export templates for different audiences (council, grants, internal)?

---

## Closing Questions

23. What's missing that would stop you from using this tomorrow?
24. What would you change about any screen you saw today?
25. On a scale of 1-5, how close is this to what you envisioned?
